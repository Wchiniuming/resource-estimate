# 计算详情模块优化计划

## 需求概述

优化计算详情模块，实现「验证结论合理性、讲清计算全链路」的核心价值。

## 优化目标

1. **UI 视觉呈现**：与上方估算模块强联动、可折叠面板、瓶颈维度默认展开
2. **内容逻辑**：完整计算链路可追溯、核心公式→分项明细→计算结果
3. **数据规范**：统一数值格式、专业术语修正

---

## 1. UI 视觉重构

### 1.1 面板结构优化

**当前状态**：三个独立板块（算力限制、显存限制、带宽限制）

**优化后**：
```
┌─────────────────────────────────────────────┐
│ [▼] 算力详情 ─ GPU需求: 1  [全部展开]     │
├─────────────────────────────────────────────┤
│ 核心公式 → 核心指标 → 分项明细           │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│ [▶] 显存详情 ─ GPU需求: 3  [全部折叠]     │
├─────────────────────────────────────────────┤
│ (折叠内容)                                │
└─────────────────────────────────────────────┘
┌─────────────────────────────────────────────┐
│ [▶] 带宽详情 ─ GPU需求: 1  [全部折叠]     │
├─────────────────────────────────────────────┤
│ (折叠内容)                                │
└─────────────────────────────────────────────┘
```

### 1.2 面板标题规范

| 维度 | 标题格式 | 示例 |
|------|----------|------|
| 算力 | 算力详情 - GPU需求: X | 算力详情 - GPU需求: 1 |
| 显存 | 显存详情 - GPU需求: X | 显存详情 - GPU需求: 3 |
| 带宽 | 带宽详情 - GPU需求: X | 带宽详情 - GPU需求: 1 |

### 1.3 全局控制按钮

- 「全部展开」：展开所有面板
- 「全部折叠」：折叠所有面板（非瓶颈维度）
- 瓶颈维度默认展开

### 1.4 信息层级规范

每个面板固定结构：
```
1. 核心计算公式（顶部，加粗）
2. 核心总指标（放大高亮）
3. 分项明细指标（网格排版）
```

### 1.5 数值规范

- 保留 2 位小数
- 大数值增加千分位分隔符
- 同一面板内统一单位层级

## 2. 内容逻辑优化

### 2.1 术语修正

| 当前术语 | 优化后术语 | 说明 |
|---------|-----------|------|
| 所需 FLOPs | 单样本推理总 FLOPs | 行业通用规范 |
| 单卡 FLOPs | 单卡峰值推理 FLOPs | 行业通用规范 |
| 低频通信 | PCIe/RDMA 跨机通信 | 明确通信类型 |
| 高频通信 | NVLink/HCCS 机内多卡通信 | 明确通信类型 |
| 单样本总FLOPs | 单样本推理总FLOPs | 统一命名 |

### 2.2 算力维度内容

```
核心公式: GPU需求 = ceil(总FLOPs需求 / 单卡峰值FLOPs)
核心指标: 单样本推理总FLOPs / 总FLOPs需求
分项明细:
- Prefill FLOPs
- Decode FLOPs
- 并发数 × 单样本 = 总需求
```

### 2.3 显存维度内容

```
核心公式: GPU需求 = ceil(总显存需求 / 单卡可用显存)
核心指标: 显存总需求 / 单卡可用显存
分项明细:
- 模型权重（核心占用，99%以上）
- 峰值激活值
- KV缓存(含QPS冗余)
```

### 2.4 带宽维度内容

```
核心公式: GPU需求 = ceil(最大带宽需求 / 单卡带宽)
核心指标: 带宽需求 / 单卡带宽 / 最大QPS
分项明细:
- 高频通信(NVLink)
- 低频通信(PCIe)
- 基于GPU数量的最大QPS
```
